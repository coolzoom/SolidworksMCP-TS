SolidWorks MCP Server 使用说明

通过 MCP 协议将 Cursor / Claude 等 AI 助手连接到 SolidWorks，实现零件创建、草图绘制、特征建模等自动化操作。

测试状态：通过（2026-06-11） 版本 3.1.3 · SolidWorks 2023
概述 环境要求 安装 启动 MCP 如何使用 常用工具 测试 测试报告 故障排除

1. 概述

SolidWorks MCP Server 是一个基于 Node.js 的 MCP（Model Context Protocol）服务，通过 winax 与 SolidWorks COM API 通信，将 CAD 操作暴露为 AI 可调用的工具。

Cursor / Claude → MCP 协议 (stdio) → solidworks-mcp-server → winax → SolidWorks COM
架构要点
  • 简单操作（≤12 参数）：直接 COM 调用
  • 复杂操作（13+ 参数）：自动生成 VBA 宏执行
  • 当前注册 86 个工具（建模、草图、工程图、导出、VBA 等）

3. 安装

3.1 一键安装（推荐）

以管理员身份打开命令提示符，进入项目目录：

cd C:\Users\hands\Documents\GitHub\SolidworksMCP-TS
install.bat --install-deps

安装脚本会自动完成：

  1. 检测 / 安装 Node 20、Python、VS Build Tools
  2. 检测 SolidWorks 版本与安装路径
  3. 同步 .env 配置
  4. npm install + 编译 winax@3.4.2
  5. 构建 TypeScript（dist/index.js）
  6. 配置 Cursor / Claude MCP

3.2 仅开发模式（无 SolidWorks）

install.bat --mock-only

3.3 验证安装

node scripts\check-winax.mjs
REM 期望输出: winax: OK

npm run build
node scripts\test-mcp-smoke.mjs --real

4. 启动 MCP

4.1 在 Cursor 中使用（推荐）

安装完成后，MCP 配置写入 %USERPROFILE%\.cursor\mcp.json。推荐配置示例：

{
  "mcpServers": {
    "solidworks": {
      "command": "C:/Program Files/nodejs/node.exe",
      "args": [
        "C:/Users/hands/Documents/GitHub/SolidworksMCP-TS/dist/index.js"
      ],
      "env": {
        "SOLIDWORKS_PATH": "C:\\Program Files\\SOLIDWORKS Corp\\SOLIDWORKS",
        "SOLIDWORKS_VERSION": "2023",
        "ADAPTER_TYPE": "winax-enhanced",
        "LOG_LEVEL": "info"
      }
    }
  }
}

启动步骤：

  1. 先启动 SolidWorks（保持运行）
  2. 完全关闭并重新打开 Cursor
  3. 打开 Settings → MCP，确认 solidworks 显示为绿色/已连接
  4. 在 Agent 对话中即可调用 SolidWorks 工具

4.2 在 Claude Desktop 中使用

配置文件路径：%APPDATA%\Claude\claude_desktop_config.json，格式与上方相同。修改后重启 Claude Desktop。

4.3 手动启动（调试）

REM 使用 dev.bat（自动刷新 PATH）
dev.bat

REM 或直接运行编译后的服务
set USE_MOCK_SOLIDWORKS=false
node dist\index.js

手动模式通过 stdio 与 MCP 客户端通信，通常仅供调试，日常使用请通过 Cursor 自动管理。

5. 如何使用

5.1 在 Cursor 对话中的用法

连接成功后，直接用自然语言描述 CAD 任务，AI 会自动选择并调用 MCP 工具。示例提示词：

你说AI 调用的工具链
「新建一个零件」create_part
「在前视基准面画一个半径 25mm 的圆」create_sketch → sketch_circle
「拉伸 10mm」create_extrusion
「导出为 STEP」export_file
「生成批量导出 VBA 脚本」generate_vba_script

5.2 典型建模流程

create_part → create_sketch → sketch_circle / add_line → create_extrusion → export_file

5.3 注意事项

  • 执行任何工具前，MCP 会自动连接 SolidWorks（需 SolidWorks 已运行）
  • 草图坐标单位为 毫米 (mm)，内部自动转换为米
  • 首次调用可能较慢（COM 连接初始化）
  • 若 MCP 面板显示红色，先检查 winax 和 Node 版本

6. 常用工具参考

工具名说明关键参数
create_part新建零件文档无
create_sketch在基准面创建草图plane: Front/Top/Right
sketch_circle绘制圆center {x,y}, radius (mm)
add_line绘制直线start, end 坐标
create_extrusion拉伸特征depth, direction
open_model打开已有文件filePath
export_file导出 STEP/STL/PDF 等format, outputPath
generate_vba_script生成 VBA 宏代码template, parameters

完整工具列表（86 个）可通过 tools/list MCP 请求或运行冒烟测试查看。

9. 故障排除

现象原因解决
npm 找不到 Cursor 终端 PATH 未刷新 运行 dev.bat 或重启 Cursor；见 scripts\ensure-path.bat
Cannot find module 'winax' winax 未编译或 Node 版本不对 install.bat --install-deps，确保 Node 20
winax 编译失败 缺少 Python / VS Build Tools 安装 Python 3.12 + VS C++ 工作负载
MCP 连接红色 服务启动崩溃 手动运行 node dist\index.js 查看错误；检查 winax
Failed to connect to SolidWorks SolidWorks 未运行 先启动 SolidWorks，再调用工具
Node 24 不兼容 winax 原生模块限制 winget install OpenJS.NodeJS.20
